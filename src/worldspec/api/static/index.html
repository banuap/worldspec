WorldSpec Studio Model the enterprise. Predict the transition. Preserve what matters.

Point WorldSpec at a GitHub repository (or local path). It surveys the code and generates a model. The compiler validates it before it is saved.

Entities (nodes) and relationships (edges) of the active model.

Paste an observed world (entities + state). Each instance is checked against its invariants.

Choose a transition and a context. The runtime builds a candidate future state and checks the preserved invariants.

Run a simulation to populate evidence, or look one up by id.
